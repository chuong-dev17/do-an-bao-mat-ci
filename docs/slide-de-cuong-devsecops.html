1. Nội dung

Giới thiệu dự án và bài toán bảo mật

Dự án hiện tại

  • Car_Dealership là hệ thống quản lý gara ô tô phát triển bằng Java Web.
  • Mã nguồn thể hiện rõ cấu trúc MVC với controller, DAO, DTO và giao diện JSP.
  • Ứng dụng có đăng nhập theo vai trò và các phân hệ nghiệp vụ tách biệt.

Vấn đề bảo mật

  • Kiểm tra thủ công chậm và dễ bỏ sót khi code thay đổi liên tục.
  • Dễ phát sinh SQL Injection, XSS hoặc lộ thông tin nhạy cảm trong mã nguồn.
  • Cần đưa SAST vào pipeline để phát hiện sớm ngay từ pull request hoặc push mới.
Slide này giới thiệu đúng project và lý do cần tự động quét bảo mật.2
2. Bài toán

Input / Output của hệ thống

Input

Mã nguồnServlet, JSP, DAO, DTO, SQL script
CI/CDWorkflow security-ci.yml trên GitHub Actions
SASTSemgrep ruleset hoặc SonarQube rules

Output

  • Cảnh báo bảo mật theo file và số dòng.
  • Phân loại mức độ: Low, Medium, High.
  • Nhóm lỗi ưu tiên: SQL Injection, XSS, Hardcoded Secrets, lỗi xác thực và session.
Đầu vào là mã nguồn và workflow, đầu ra là báo cáo có thể truy vết.3
